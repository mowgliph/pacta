# Flujo de la Aplicación PACTA

## 1. Flujo de Entrada

### Modo Público (Sin autenticación)
1. Landing page con información general
2. Dashboard público con datos de demostración
   - Estadísticas generales
   - Lista limitada de contratos
   - Gráficos básicos
3. Acceso restringido a funciones premium

### Modo Autenticado
1. Login con credenciales
   - Usuario RA (desarrollador)
   - Usuario Admin (cliente)
   - Usuarios adicionales creados
2. Redirección al dashboard completo

## 2. Dashboard Principal

### Sección Superior
- Estadísticas en tiempo real
- Contratos próximos a vencer
- Indicadores de rendimiento

### Panel de Acciones Rápidas
- Crear nuevo contrato
- Agregar suplemento
- Ver estadísticas
- Acceder a backups

### Actividad Reciente
- Últimas modificaciones
- Suplementos agregados
- Contratos actualizados

## 3. Gestión de Contratos

### Listado Principal
- Tabla virtualizada de contratos
- Filtros avanzados
  * Por tipo (Cliente/Proveedor)
  * Por estado
  * Por fecha
- Búsqueda global

### Detalles de Contrato
- Información general
- Documentos adjuntos
- Historial de suplementos
- Acciones disponibles
  * Editar contrato
  * Agregar suplemento
  * Ver documentos
  * Gestionar permisos

## 4. Sistema de Notificaciones

### Tipos de Alertas
- Contratos próximos a vencer
- Nuevos suplementos
- Cambios en contratos
- Alertas de sistema

### Canales de Notificación
- Notificaciones en app
- Correos electrónicos
- Notificaciones de sistema

## 5. Gestión de Usuarios

### Roles Principales
- RA (Desarrollador)
  * Acceso total al sistema
  * Gestión de usuarios
  * Configuración avanzada
- Admin (Cliente)
  * Gestión de contratos
  * Gestión de suplementos
  * Reportes y estadísticas
- Usuarios Personalizados
  * Permisos granulares
  * Acceso restringido

## 6. Sistema de Backups

### Backups Automáticos
- Programación configurable
- Compresión de datos
- Verificación de integridad

### Gestión de Archivos
- Almacenamiento estructurado
- Versionado de documentos
- Purga automática

## 7. Flujo de Datos

### Frontend → Backend
1. Solicitudes autenticadas con JWT
2. Validación de permisos
3. Procesamiento de datos
4. Respuesta formateada

### Base de Datos
1. SQLite para almacenamiento local
2. Prisma para gestión de datos
3. Migraciones versionadas
4. Backups periódicos
